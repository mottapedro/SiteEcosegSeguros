# WEB-003 — Content Strategy

**Marca:** EcoSeg Seguros

**Identificador:** WEB-003

**Categoria:** Web / Conteúdo

**Status:** Ativo

**Versão:** 1.0

Consolida, para uso prático de quem escreve conteúdo (copy de página, CTAs, e-mails, artigos), as regras já estabelecidas em `BD-002` (Voice & Tone) e `DES-003` (Interaction Language). Este documento não introduz regras novas — organiza o que já existe em formato de referência rápida.

---

## 1. Vocabulário Preferencial (per `BD-002 §5`)

Proteção, tranquilidade, continuidade, planejamento, prevenção, consultoria, confiança, parceria, patrimônio, responsabilidade, solução personalizada, gestão de riscos, atendimento especializado.

## 2. Expressões Proibidas (per `BD-002 §5`, `DES-003 §2`)

- Urgência/medo: "última oportunidade", "compre agora", "sem isso sua empresa está condenada", "oferta por tempo limitado".
- Pressão: "aproveite", "garanta já", "não perca tempo".
- Exagero emocional: exclamação para criar excitação artificial, emojis como substituto de clareza.

## 3. Estrutura de Headline/Subheadline

Padrão estabelecido em `DES-007` (Home Experience): headline nomeia o núcleo ou a etapa da narrativa (patrimônio, adversidade, compreensão, decisão, proteção, progresso — `DES-001 §2`); subheadline concretiza com exemplos reais, nunca abstrações soltas.

Exemplos já escritos (`DES-007`):

- "Um patrimônio não é só o que você tem. É o que você ainda está construindo." (Capítulo 1)
- "Antes de qualquer cobertura, mapeamos o seu contexto." (Capítulo 3)

## 4. CTAs (per `DES-003 §5`)

Nomeiam a ação real: "Solicitar diagnóstico", "Entender [Produto]", "Agendar consultoria", "Falar no WhatsApp". Nunca urgência ("Comece agora!") nem CTA que esconde o destino real ("Saiba mais" para um formulário de contato).

## 5. Fluxo de Atendimento na Escrita

Todo conteúdo deve poder ser lido como uma instância de Ouvir → Compreender → Orientar → Recomendar (`BD-002 §6`) — nunca abrir um texto recomendando um produto antes de demonstrar compreensão do contexto do leitor.

## 6. Governança de Conteúdo por Tipo de Página

| Tipo de página                     | Fonte de regras primária                                                                |
| ---------------------------------- | --------------------------------------------------------------------------------------- |
| Home, páginas institucionais       | `DES-007` (narrativa), este documento (vocabulário/CTA)                                 |
| Páginas de produto (Soluções)      | `BD-002 §3` (portfólio), `DES-002` (arquitetura), este documento                        |
| Formulários e mensagens de sistema | `DES-003 §§4–9` (regras completas de microcopy/erro/sucesso)                            |
| Conteúdos/Blog (futuro)            | `DES-002` (seção Conteúdos), este documento — sem regras adicionais ainda especificadas |

## 7. Teste de Aceite

Antes de publicar qualquer conteúdo novo: passa no vocabulário (§1), não usa nenhuma expressão proibida (§2), o CTA nomeia a ação real (§4), e o texto seria compreendido por um síndico de condomínio sem jargão (`VIS-001 §1.4`).

## Rastreabilidade

Depende de: `BD-002 §5`/`§6`, `DES-001 §2`, `DES-003`, `DES-007` — este documento consolida, não substitui nenhum deles.

Afeta: qualquer copy futura do site, e-mails transacionais, e conteúdo de `Conteúdos`/Blog quando implementado.

Itens em aberto: regras específicas para conteúdo de Blog/SEO ainda não especificadas — apenas herdam as regras gerais deste documento por enquanto.
